# SecureNet AI — ML-Powered Network Intrusion Detection System

> An AI-powered system that classifies network traffic as normal or anomalous, and explains *why* — built on IBM watsonx.ai AutoAI and IBM Granite, deployed as a live web application.

**🔗 Website Link:** [https://network-intrusion-detection-14k5.vercel.app/](https://network-intrusion-detection-14k5.vercel.app/)

---

## 📌 Overview

Traditional intrusion detection systems rely on static, rule-based signatures that struggle to keep up with evolving attack patterns and generalize poorly to novel threats. Manual monitoring of network logs is slow, error-prone, and doesn't scale with modern traffic volumes.

**SecureNet AI** addresses this by using machine learning to classify network connections as **normal** or **anomalous** based on connection-level features (protocol, service, byte counts, error/rejection rates, and host-level statistics), and pairs every prediction with a **plain-English explanation** generated by IBM Granite — turning a raw classification into an actionable security insight instead of a black-box label.

Built for **AICTE 2026 — Problem Statement No. 40: Network Intrusion Detection**, under IBM SkillsBuild for University Engagement.

## 🎯 Problem Statement

Organizations and network administrators face increasing difficulty identifying malicious or anomalous traffic among massive volumes of legitimate connections. This project designs, builds, and evaluates a machine learning model that classifies network connections as normal or anomalous, supporting security teams in detecting intrusions early and reducing manual monitoring effort.

---

## 🧠 Approach

1. **Data preprocessing** — encoding categorical features (protocol, service, flag), scaling numeric features
2. **Model training** — IBM watsonx.ai **AutoAI** automatically evaluated multiple algorithms (Random Forest, XGBoost, LGBM, Logistic Regression, Snap Decision Tree, and others), performing algorithm selection and hyperparameter optimization
3. **Model selection** — **Snap Decision Tree Classifier** selected as the best-performing pipeline
4. **Deployment** — model deployed as a live REST API via **Watson Machine Learning** on IBM Cloud
5. **Explainability layer** — **IBM Granite** (`granite-3-8b-instruct`) integrated to generate human-readable explanations for each flagged connection
6. **Application layer** — Flask backend bridges the frontend to IBM Cloud services; responsive dashboard deployed on Vercel

```
Raw Traffic Features → AutoAI Model (WML) → Prediction + Confidence
                                          ↓
                              IBM Granite → Plain-English Explanation
                                          ↓
                                  Dashboard Display
```

## ⚙️ Running Locally

### Backend
```bash
cd backend
pip install -r requirements.txt
cp .env.example .env   # fill in your IBM Cloud credentials
python app.py
```

### Frontend
Open `frontend/dashboard.html` in your browser (or serve via a local static server), ensuring `BACKEND_URL` in the dashboard script matches your running backend (default: `http://localhost:5000`).

### Environment Variables (`backend/.env`)
```
IBM_API_KEY=
WML_DEPLOYMENT_URL=
WML_URL=
WATSONX_PROJECT_ID=
```
